Zero-Inflated Count Models

This procedure estimates a mixture model for count data with a point mass at zero. Predictions can be made after estimation using an in-memory workspace, or later from a saved workspace.

When predicting, the model specification is taken from previous estimation. Do not enter model parameters in that mode. The Options dialog is only used for estimation. The Save subdialog has settings that apply to both modes.

Model Source Choose whether to estimate a model or to make predictions using a previously estimated model. A dataset name for the output must be specified in the Save subdialog if making predictions.

Specify the file containing the saved model if predicting from a model file. If using the retained workspace, the Save subdialog must have specified that the workspace should be retained, and no other R-based procedure may have been run since the model was estimated. Only the most recently estimated model in the workspace is available for use in prediction

Dependent Select the dependent variable. It is expected to contain counts.

Count Model Predictors Specify the regressors for the count model.

Use count predictors for zero model Check this box to use the same predictors for both the count model and the zero model, and leave the Zero Model Predictors field empty. If the box is not checked, you can specify a different set of predictors for the zero model. In that case, if no predictors are specified, the zero model is the same for all cases. That is, the only predictor is the constant term.

Categorical predictors are automatically treated as factors.

Zero Model Predictors Specify any regressors for the binary 0/1 model unless the box above was checked.

ID An ID variable can be specified to be saved in the estimation and prediction dataset.

Count Model and Zero Model Offsets You can specify offset variables for the models. The Use count predictors checkbox does not affect these settings.

Options

Count Model Distribution Choose the distribution for the count model to use with a log link. The geometric distribution is the negative binomial distribution with size (number of failures) parameter 1.

Link function for Zero Model Choose the appropriate link function. The distribution will be binomial.

Missing Values Choose whether to omit cases with missing values or to stop the procedure.

Method for Finding Starting Values Choose whether to use a generalized linear model or the EM algorithm to calculate the starting values for the iterative estimation procedure.

Solution Method A variety of solution methods are available. The default, however, generally works well.

Maximum Number of Iterations You can set a limit on the number of iterations.

Convergence Tolerance You can control the definition of convergence here.

Save

The Save subdialog determines what output is saved for future use.

Output Dataset This specifies the output dataset name. For estimation, the dataset will contain the fitted values and residuals. For prediction, it will contain the predicted values and is required.

The dataset name must not already be in use.

Workspace Action Specify whether to clear the in-memory workspace after the procedure or to retain it for prediction calculations to follow in the same session.

Save Workspace specifies whether to save the estimated model to a file. It only applies to estimation mode. A saved workspace can be loaded and used for calculating predictions in the future.

Additional Features

This dialog generates syntax for the STATS ZEROINFL extension command. To display syntax help for this command, run the following syntax:

STATS ZEROINFL /HELP.
